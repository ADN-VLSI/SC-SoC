## Overview

The UART Receiver (`uart_rx`) is a hardware module that converts incoming serial data into parallel bytes. It works without a shared clock by detecting a start bit, then sampling each data bit at the correct time using an oversampling clock. It can also check parity and verify the stop bit before outputting valid data.

This module is typically used inside a larger UART system, where it connects to a clock divider and a receive FIFO.

## Functional Specification

The UART Receiver handles the receiving side of UART communication. It uses a baud-rate clock to read incoming serial data and converts it into one byte at a time.

Main features:

* Detects the start bit and ignores false triggers
* Supports 8-bit data 
* Optional parity checking (even or odd)
* Supports 1 or 2 stop bits
* Generates a parity error signal if needed
* Provides a data valid signal to pass data to a FIFO or other logic

## Parameters

The UART Receiver uses the following parameters for configuration:

| Parameter   | Description                          | Default Value |
| ----------- | ------------------------------------ | ------------- |
| DATA_BITS   | Number of data bits per frame        | 8             |
| OVERSAMPLE  | Oversampling factor for bit sampling | 16            |
| PARITY_EN   | Enable parity checking (optional)    | 0             |
| PARITY_TYPE | 0 = Even parity, 1 = Odd parity      | 0             |
| STOP_BITS   | Number of stop bits per frame        | 1             |

### State Descriptions
 
| State         | Description                                                                                       |
| ------------- | ------------------------------------------------------------------------------------------------- |
| IDLE          | Waits for a falling edge on the filtered `rx_i` line indicating the start of a new frame          |
| START         | Waits 8 oversampling ticks (half bit period) then re-samples; advances if LOW, aborts if HIGH     |
| DATA_BITS     | Samples `rx_i` at the center of each of the 8 data bit windows and shifts into the shift register |
| PARITY        | If `parity_en_i` is set, samples the parity bit and compares against the computed parity value    |
| STOP          | Samples the stop bit; if HIGH (valid), asserts `data_valid_o` for one cycle; returns to IDLE      |  

## Baud Rate and Oversampling
 
The `uart_rx` module operates on a pre-divided clock generated by the `clk_div` module. The effective baud rate and oversampling factor are set by configuring the clock divider register.
 
### Baud Rate Divider Formula
 
```
CLK_DIV = AXI4L_Clock / (Baud_Rate x OVERSAMPLE)
 
Example (115200 baud, 100 MHz clock, 16x oversampling):
  CLK_DIV = 100,000,000 / (115,200 x 16) = 54.25  =>  54
```
### Sampling Point Alignment
 
With 16× oversampling, each bit period contains 16 clock ticks. The receiver centers the sample point by:
 
1. Detecting the falling edge of the start bit.
2. Waiting 8 ticks (half period) to align to the center of the start bit.
3. Sampling every 16 ticks thereafter for each data, parity, and stop bit.

### Parity Error
 
When `parity_en_i` is asserted, the module computes the parity of the 8 received data bits and compares it to the received parity bit according to `parity_type_i`:
 
| `parity_type_i` | Expected Behavior                                         |
| --------------- | --------------------------------------------------------- |
| 0 (Even)        | Total number of 1s in data + parity bit must be **even** |
| 1 (Odd)         | Total number of 1s in data + parity bit must be **odd**  |
 
If a mismatch is detected, `parity_error_o` is asserted HIGH for one clock cycle alongside `data_valid_o`. The byte is still presented on `data_o`, allowing the host to decide whether to discard or accept it.
